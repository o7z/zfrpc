# 使用场景

## 场景 1：本地开发环境

开发者在 Windows 本地开发，需要临时开放内网端口给外部访问。

**痛点**：每次修改 frpc 配置需要编辑文件、重启进程。

**zfrpc 解决**：Web 界面修改配置，一键生效。

## 场景 2：测试环境管理

开发者在 Linux 测试服务器上运行 frpc，需要频繁调整配置。

**痛点**：需要 SSH 到服务器修改配置，操作繁琐。

**zfrpc 解决**：远程访问 zfrpc Web 界面，无需 SSH。

## 场景 3：多配置切换

用户有多个 frpc 配置（开发、测试、生产），需要快速切换。

**痛点**：手动备份/恢复配置文件。

**zfrpc 解决**：配置列表管理，一键切换。

## 场景 4：开机自启管理

用户希望服务开机后自动运行 frpc。

**痛点**：不同平台配置方式不同（Windows 任务计划 / Linux systemd）。

**zfrpc 解决**：统一界面配置，自动处理平台差异。
