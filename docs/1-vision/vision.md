# 为什么做这个项目

## 背景

frp 是一个优秀的内网穿透工具，但日常管理 frpc 存在以下痛点：

1. **配置修改繁琐**：需要手动编辑 toml/ini 文件，容易出错
2. **进程管理不便**：启动/停止/重启需要命令行操作
3. **日志查看困难**：日志输出到终端或文件，不便于实时监控
4. **开机自启配置复杂**：不同平台（Windows/Linux）需要不同的配置方式

## 目标

提供一个带有 Web 界面的本地服务，让用户能够：

- 通过浏览器直观管理 frpc 配置
- 一键启动/停止/重启 frpc 进程
- 实时查看 frpc 运行日志
- 配置本服务和 frpc 的开机自启

## 设计原则

- **轻量**：安装简单，资源占用小
- **跨平台**：Windows + Linux 优先
- **零依赖**：用户安装后无需额外配置即可使用
- **安全**：可选密码保护，防止未授权访问
